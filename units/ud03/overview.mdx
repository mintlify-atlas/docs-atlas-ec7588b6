## BIOS vs UEFI

<Note>
  UEFI está reemplazando a BIOS como el firmware estándar en ordenadores modernos.
</Note>

### Comparación Rápida

| Aspecto | BIOS | UEFI |
|---------|------|------|
| **Año de introducción** | 1975 | 2005 |
| **Interfaz** | Texto | Gráfica y texto |
| **Tamaño máximo de disco** | 2TB (MBR) | 9ZB (GPT) |
| **Particiones** | 4 primarias | 128 particiones |
| **Tiempo de arranque** | Más lento | Más rápido |
| **Secure Boot** | No | Sí |
| **Modo** | 16-bit | 32/64-bit |

## Proceso de Instalación

### Pasos Generales

<Steps>
  <Step title="Requisitos Previos">
    - Verificar requisitos del sistema
    - Descargar imagen ISO
    - Preparar medio de instalación (USB/DVD)
    - Hacer backup de datos importantes
  </Step>
  
  <Step title="Configurar BIOS/UEFI">
    - Configurar orden de arranque
    - Habilitar/deshabilitar Secure Boot según necesidad
    - Verificar modo de arranque (Legacy/UEFI)
  </Step>
  
  <Step title="Arrancar desde Medio">
    - Iniciar desde USB/DVD
    - Acceder al menú de instalación
  </Step>
  
  <Step title="Particionado">
    - Elegir esquema de particiones
    - Formatear particiones
    - Asignar puntos de montaje (Linux)
  </Step>
  
  <Step title="Instalación">
    - Copiar archivos del sistema
    - Configurar usuarios
    - Instalar gestor de arranque
  </Step>
  
  <Step title="Configuración Inicial">
    - Actualizaciones
    - Drivers
    - Configuración de red
  </Step>
</Steps>

## Modos de Arranque Especiales

### Windows

<Accordion title="Modo Seguro">
  Arranca Windows con drivers y servicios mínimos para diagnóstico y reparación.
  
  **Variantes:**
  - Modo Seguro
  - Modo Seguro con Funciones de Red
  - Modo Seguro con Símbolo del Sistema
</Accordion>

<Accordion title="Entorno de Recuperación (WinRE)">
  Conjunto de herramientas para reparar problemas de arranque.
  
  **Herramientas:**
  - Restaurar sistema
  - Reparación de inicio
  - Símbolo del sistema
  - Desinstalar actualizaciones
</Accordion>

### Linux

<Accordion title="Modo Recovery">
  Modo de recuperación con acceso root para reparaciones.
  
  **Usos:**
  - Reparar sistema de archivos
  - Resetear contraseñas
  - Actualizar GRUB
  - Reparar paquetes rotos
</Accordion>

<Accordion title="Single User Mode">
  Modo de usuario único para mantenimiento del sistema.
</Accordion>
